# 從 Discord 年齡驗證談起：我們為什麼自架 Matrix

![從 Discord 年齡驗證談起：我們為什麼自架 Matrix，以及如何兼顧隱私與社群品質](https://assets.anoni.net/blog/2026-discord-matrix-statement.png){style="border-radius: 5px;"}

2026/02/09，[Discord 宣布在全球推出「teen-by-default」設定](https://discord.com/press-releases/discord-launches-teen-by-default-settings-globally){target="_blank"}，並強化年齡驗證機制（台灣報導見 [巴哈姆特 GNN：Discord 宣布 3 月起全球實施「青少年保護」措施](https://gnn.gamer.com.tw/detail.php?sn=299872){target="_blank"}）。新用戶與既有用戶將預設進入以青少年為考量的體驗，若要解除部分內容過濾或存取年齡分級空間，需透過臉部年齡估計或提交證件等方式完成驗證。Discord 強調這是為了青少年安全與 Safer Internet Day 的承諾，也提到會引入「年齡推論模型」在背景協助判斷帳號是否屬於成人。

我們無意批評 Discord 的初衷，青少年保護與合規是嚴肅議題。但這類機制也代表一件事：**大型平台會需要更多個人資料與行為訊息來「分類」使用者**。不論是臉部影像、證件、或是演算法推論，最終都是把「你是誰、你幾歲、你在哪裡」交給平台與其合作廠商。對許多只想安心聊天、玩遊戲、或參與專案的人來說，這可能是可接受的取捨。對另一群人來說，卻會開始思考：有沒有別的路？

## 我們真正在意的是：誰決定規則、誰拿走資料

商業聊天平台有其遊戲規則：服務條款、產品方向、哪些資料被保留、演算法如何運作，多半由公司與股東決策驅動，一般使用者難以參與，也難以完全掌握自己的資料如何被使用。這不是「誰比較壞」的問題，而是**誰有權決定**的問題。

匿名網路社群 anoni.net 選擇的是另一條路：**自架 Matrix 家伺服器**（homeserver）。我們使用 [tuwunel](https://github.com/matrix-construct/tuwunel){target="_blank"} 這套以 Rust 撰寫、高效能且符合 Matrix 規範的 homeserver，在 `im.anoni.net` 上承載社群討論與 2026 主題協作。伺服器設定、紀錄保存策略、頻道規則，由維運者與社群成員一起決定，範圍小、可預期、也相對透明。我們的核心很明確：圍繞**網路自由、匿名網路、隱私實踐**，而不是「任何人進來隨便聊什麼都好」。這裡是一個有主題、有共識的工作空間。

## 為什麼我們選擇自架 Matrix（tuwunel + im.anoni.net）

[Matrix](https://matrix.org/) 是一種開放、去中心化的即時通訊協定和生態系，讓全球不同的平台能彼此安全溝通、互相串聯。每個人都可以自架屬於自己的 Matrix homeserver（家伺服器），自行決定伺服器規則、紀錄保存方式與驗證政策。因此，我們選擇自架 `im.anoni.net`，並以 [tuwunel](https://github.com/matrix-construct/tuwunel){target="_blank"} （一套以 Rust 開發的高效能 Matrix homeserver）來服務社群協作。

目前「匿名網路社群」在帳號申請上，採**註冊碼註冊制度**，你需要來信 `whisper@anoni.net` 或透過其他可信任的管道取得註冊碼，再以註冊碼在 Matrix 上完成註冊。**註冊當下**不會要求你提供信箱、帳號名稱可自選，伺服器端也不保存可辨識身份的註冊紀錄。註冊碼**非一次性（可多次共用）**，無法從註冊碼反推是誰在何時註冊。也就是說，取得註冊碼時會有一次來信聯絡，但**在系統內註冊與使用時**不綁定信箱或實名，我們盡量把「誰是誰」的資訊壓在營運所需的最小範圍。

選擇自架與這樣的註冊流程，最直接的好處是：

- **紀錄最少、追蹤最少**：我們盡量壓縮「誰在什麼時候做了什麼」的保存範圍，能刪就刪。
- **完全不要求實名或驗證**：不會強制上傳證件、也不用手機、信箱，只需使用註冊碼即可註冊。
- **為敏感議題、匿名研究而設計的彈性政策**：無論你是參與匿名網路、審查觀測或其他高敏感度範疇，都不用擔心帳號曝光風險，我們會因應社群需要逐步調整相關政策。

所以，**我們選擇 Matrix，不是單純因為 Discord 不夠好**，而是大型平台的政策必須服膺於商業與普羅大眾，我們則需要一個能自己決定規則、對用戶最小化資訊收集、並高度重視匿名性與隱私的空間。

## 在隱私與社群品質之間：為什麼目前採「申請制」帳號

因此，**Matrix（im.anoni.net）與 [Cryptpad](https://cryptpad.anoni.net/) 帳號都需來信 `whisper@anoni.net` 或透過其他可信任的管道申請註冊碼**，由我們回覆註冊方式或註冊碼後，你再於客戶端完成註冊。我們不開放任何人一鍵註冊，以兼顧濫用防護與上述「註冊階段不存個資」的設計。

這裡也簡單說明什麼是 Cryptpad：**Cryptpad 是一個強調端對端加密、隱私保護的線上協作平台**，可用於多人共筆、即時筆記、文件協作、待辦列表、白板等，內容全程加密，只有參與的成員才能存取與查看。不像 Google Docs 等主流雲端工具，Cryptpad 沒有伺服器端解密權限，也無需信任平台營運者，特別適合有機密、隱私需求的協作場景。「匿名網路社群」自架 Cryptpad 服務，就是為了讓社群在討論、共筆、協作時，能最大限度確保資訊不被第三方蒐集或監控。

這樣採用申請制的原因很務實：

- **避免系統被惡意利用**：如果讓大家自由註冊，很容易就會有自動化垃圾帳號或不懷好意的人湧入，對我們這種小規模自架的服務來說，管理和維護的壓力會變得很大。
- **讓每個進來的人都了解這裡的定位**：申請的人知道，這個空間是專門討論像「個人隱私指引」、「Tor 中繼節點在校園的推動」、「匿名支付」這幾個主題，不是那種什麼都可以閒聊的地方。會主動寫信申請的朋友，也多半對這些議題已經有一定的興趣和認識。
- **我們這樣做是為了維護品質，而不是為了排除誰**：我們不會要求你真實姓名，也不會要你上傳證件資料，只是需要你多花一點時間寫信申請註冊碼、簡單說明你有興趣參與什麼。只要你對這些主題有興趣，都很歡迎聯絡我們加入這個空間。

帳號申請與使用方式請見本站 [溝通與協作工具](../../about/community/communication-tools.md)。

## 對正在使用 Discord 的朋友說的話

我們不希望大家覺得「非得二選一」：很多人需要留在 Discord 上的社群、遊戲、專案，這完全可以理解。我們想說的是：**當你需要談敏感議題、需要更高隱私與可預期的環境時，這裡有一個小而穩定的 Matrix 空間可以使用**。

- 如果你想深入參與 2026 三個主題（個人隱私指引、Tor Relay 校園建立、匿名支付），或是實作匿名網路相關的實驗，歡迎優先在 [Matrix、Cryptpad](../../about/community/index.md) 參與。
- 若你只是關心消息、偶爾旁聽，維持原本的平台也沒關係。我們會持續在文件網站與活動中分享進度，你隨時可以再決定要不要參與我們。

anoni.net 的社群參與大門一直開著，差別只在於、我們選擇用「**誰有權決定規則、誰拿走資料**」來思考這條路，並用自架 Matrix 與申請制帳號，在隱私與品質之間取得我們能接受的平衡。
