# :material-gavel: 治理章程

!!! warning "草案徵求共識中"

    這份治理章程目前還是草稿，內容尚未正式取得社群夥伴的共識，隨時可能調整。歡迎一起來討論社群的未來，可以在 Matrix 公開 room 或 GitHub Issue 提出想法。具體的修訂流程見下方 [章程修訂](#章程修訂)。

社群運作到一定規模後，需要把「如何做決定」這件事寫下來，避免每次重新爭論基本原則。這份治理章程說明 anoni.net 社群的角色定義、決策方式、爭議處理與行為準則。內容是一份活文件，重大調整都會在 Matrix 公開討論後才提交修訂。

讀這份章程的兩種方式：第一次參與的成員可以略讀「角色」與「行為準則」兩節，掌握加入社群的基本期待。資深成員、提案者、需要處理爭議的核心成員，建議完整讀過一遍。

## 適用範圍

本章程適用於以下範疇的協作與互動：

- anoni.net 文件站（zh-TW、zh-CN、en）
- 自架的 Matrix 家伺服器 `im.anoni.net`、Cryptpad、Etherpad、Send、SearXNG、Formbricks
- 以 anoni.net 名義舉辦的工作坊、小聚、研討會場次
- 與 EFF、Tor Project、OONI 等國際組織合作時的對外代表
- 社群相關 GitHub repo 的 PR、Issue、Discussion

不涵蓋成員以個人身分在外部平台的言行，但在 anoni.net 場域內的言行需符合本章程。

## 成員角色

社群維持低度形式化的角色制度，避免層級造成新人疏離。實際的權限差別主要在帳號層面（誰能合 PR、誰能審 Matrix 帳號申請），不在發言權重上。

### 核心成員

長期投入維運、累積足夠技術與社群信任的成員。負責：

- 自架服務的維運與帳號審核
- GitHub repo 的合併權限
- 對外代表 anoni.net 與國際組織交涉
- 章程修訂提案的最終把關
- 處理違反行為準則的爭議

成員資格由現有核心成員共識決同意後加入，目前沒有公開申請流程。

### 貢獻者

具體有產出的成員，包括寫文章、翻譯、寫程式、籌備活動、做設計、做研究等。在 Matrix Public Space 內的相關 room 活躍參與。

加入方式：到 [如何參與與認領主題](./how-to-contribute.md) 認領主題、提 PR，或參與活動籌備。沒有正式門檻。

### 觀察員

訂閱電子報、加入 Public Space 但暫時沒有具體產出的成員。同樣歡迎，社群運作偏 async，不要求每個人都活躍。

### 訪客

未加入 Matrix 但讀文件、看 blog、寄信來討論的人。寄到 <whisper@anoni.net> 的訊息會由核心成員輪值處理。

## 決策方式

### 共識決為主

社群以共識決為基本運作方式。這裡的「共識」是相關成員都不反對的狀態，不要求全員同意。具體做法：

1. 提案者在 Matrix 對應 room 提出，附上脈絡與選項
2. 給予至少 3 天讓有意見的成員回應
3. 沒有反對意見即視為通過
4. 有反對意見就進入討論，討論到雙方都能接受的版本

這個流程適合大多數的內容、流程、設計決策。

### 投票（例外狀況）

只有當共識決明顯卡關、或時間壓力下需要快速結論時，才走投票。投票條件：

- 由核心成員至少兩人發起
- 投票對象：所有貢獻者（最近 6 個月內有具體產出的成員）
- 投票方式：Matrix Poll 公開或 Cryptpad form 半匿名
- 通過門檻：簡單多數（過半）
- 重大議題（章程修訂、核心成員加入或退出）需 2/3 多數

實務上目前社群幾乎沒走過投票，多數情況共識決就能解決。

### 提案流程

| 影響範圍 | 提案管道 | 決策方式 |
|---|---|---|
| 單篇文章內容、翻譯選詞 | GitHub PR / Matrix room | review 共識 |
| 結構性變動（改 nav、搬檔、改命名） | Matrix room 先討論再 PR | 共識決，3 天緩衝 |
| 自架服務新增、移除、政策變更 | Matrix 核心 room + 公開 room 並行 | 核心成員共識決 |
| 章程、行為準則修訂 | GitHub Issue + Matrix 雙管 | 共識決，至少 7 天緩衝 |
| 對外公開立場（聲明、抗議、支持書） | Matrix 公開 room | 核心成員 + 貢獻者多數同意 |

## 爭議處理

爭議分兩類：對事的爭議（如何做某件事）與對人的爭議（互動衝突、行為準則違反）。

### 對事的爭議

走一般的提案流程：在 Matrix 提出歧見、列出雙方論點、找共識。沒有共識就投票。技術性爭議可以保留分歧、各自實驗，過一段時間用結果再評估。

### 對人的爭議

涉及成員之間的互動衝突，依嚴重程度分流：

1. **輕微誤解**：相關成員私下溝通即可
2. **明確違反行為準則**：受影響者可向核心成員回報，核心成員協助處理（私下對話、公開警告、暫時隔離）
3. **嚴重事件**（騷擾、跟蹤、人身威脅）：核心成員會立即移除違規成員的存取權限，事後在公開 room 簡述處置（不揭露受害者身分），並依需要協助受害者報警或聯繫援助資源

涉及核心成員自身的爭議，由其他核心成員處理，必要時邀請外部仲裁。

## 行為準則

### 互動原則

- **互相尊重**：不同背景、技術熟悉度、政治立場的成員一視同仁
- **對事不對人**：批評想法、不批評身分
- **積極預設**：先假設對方的提議出自善意，再討論細節
- **資訊透明**：相關決策盡量在公開 room 進行，私下協調事後同步到公開
- **新人友善**：對基本問題保持耐心，把複雜的脈絡拆解清楚

### 合法前提

所有協作與討論以合法用途為前提，社群不協助也不指導以下行為：

- 洗錢、規避稅務、規避經濟制裁
- 騷擾、跟蹤、人身威脅、未授權入侵
- 散布兒少性剝削內容、人口販運、毒品交易
- 在他國對該國公民進行情報蒐集或干預內政

匿名工具與隱私保護工具會被各種人使用，社群推廣的對象是新聞媒體、公民團體、研究者、需要保護自己的個人。如果發現協作對象有上述違法意圖，協作會立即終止。

### 機敏資訊處理

- 涉及個人資料、未公開研究、敏感受害者身分的內容，走 [上傳機敏資訊流程](./upload-sensitive.md)
- 不要在公開 room 直接貼受害者個資、未公開的爆料原文
- 引用敏感報導時保留原作者來源、不擅自摘要敏感細節

### 對外言行

代表 anoni.net 對外發言（媒體訪問、活動致詞、社群帳號）需要：

- 避免發表非經社群討論的政治立場
- 對社群成員私下意見的引述需取得當事人同意
- 與國際組織（EFF、Tor Project、OONI）交涉前先在 Matrix 同步

成員以個人身分對外發言時，註明「個人立場，不代表社群」。

## 違反處置

違反行為準則的處置由核心成員依個案決定，可能的處置由輕至重：

1. **私下提醒**：核心成員私訊提醒，不留公開紀錄
2. **公開警告**：在相關 Matrix room 公開記錄違規行為
3. **暫時隔離**：暫停 Matrix 帳號、Cryptpad 存取、GitHub 權限數天到數週
4. **永久移除**：撤銷所有自架服務的帳號、合作關係、社群相關活動的參與資格

處置會考量違規嚴重性、是否累犯、當事人態度。重大事件的處置決定會在公開 room 簡述（不揭露受害者身分）。

## 章程修訂

本章程是活文件。任何人都可以提案修訂，流程：

1. 在 GitHub 開 Issue 描述要改什麼、為什麼
2. 同步在 Matrix 公開 room 通知
3. 至少 7 天討論期
4. 達共識後由核心成員合進 main，並在 blog 簡短說明異動

緊急狀況（例如新型騷擾手法出現、需要立即更新處置原則）可由核心成員臨時更新，並在 7 天內回到正規程序檢討。
